AURORA 운영 자동화

프로젝트 배경

ML 모델의 실험 성능과 실운영 성과 사이에는 큰 격차가 존재합니다. 백테스트에서 우수한 전략이 라이브 환경에서 실패하는 근본 원인은 모델 자체가 아니라 운영 전환 과정의 검증 부재입니다. AURORA는 이 격차를 줄이기 위해 실험→검증→운영을 분리하고, 각 단계에 자동화된 게이트를 배치했습니다.

핵심 인사이트
현상 모델 성능 수치만 믿고 운영 전환 시, run 단위 추적 키 누락으로 장애 원인 분석에 수 시간 소요
조치 run_id → pack_id → decision_uid 계층적 추적 체계 도입 + 4단계 승격 프로토콜 강제
결과 실험-운영 단절 해소, 장애 원인 분석 시간 단축, 무중단 전략 교체 가능

핵심 교훈

"잘 맞는 모델"보다 "안전하게 운영되는 시스템"

After 4단계 승격 + 20개 자동 검증 + 추적 키 체계 → 무중단 전략 교체
교훈 PM/운영 관점에서 실험-검증-승격-회복까지 닫힌 루프를 만드는 것이 실제 성과로 이어진다
