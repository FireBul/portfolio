Teflon Inspection Automation (Private Deep Summary)

민감정보를 제외한 현장 운영·구조 중심 요약

핵심: 비전 기반 검사 자동화 운영 성과: 현장 적용성 향상 특징: 모델 + UI + 배포 흐름 통합

프로젝트 배경

검사 자동화는 모델 정확도 외에도 카메라 연결, 운영자 인터페이스, 배포 안정성이 함께 갖춰져야 실제로 돌아갑니다. 이 프로젝트는 “실험 성공”보다 “현장 동작”을 목표로 설계했습니다.

핵심 문제

  • 카메라 소스 다양성(USB/IP/RTSP)으로 인한 연결 불안정
  • 운영자가 즉시 판단하기 어려운 UI/알림 구조
  • 업데이트 시 현장 중단 리스크

데이터 규모 & 학습/검증 깊이 (공개 가능 범위)

  • 코드베이스 관측 규모: Python 파일 약 22개, 문서 약 11개(로컬 스캔 기준)
  • 운영 자산: 설정 파일/현장 배포 가이드/업그레이드 블루프린트 포함
  • 학습 깊이: 모델 정확도 + 현장 운영성(연결/알람/업데이트) 복합 검증
  • 검증 관점: 데모 모드와 실운영 모드를 분리해 리스크를 낮춘 배포 흐름 설계

※ 원본 학습 데이터/고객 현장 식별정보/민감 운영 파라미터는 보안상 비공개입니다.

서버/운영 상태 관점 (포트폴리오용)

  • 현장 실행은 bootstrap/start 스크립트 기반으로 표준화
  • 설정·카메라·알람 모듈을 분리해 문제구간 진단 속도 향상
  • 업데이트 전후 체크리스트 운영으로 다운타임 리스크 완화
